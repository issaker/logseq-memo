5. 任务目标：测试体系审查与核心测试补强 (Test Infrastructure & Coverage)
    - 你是一位 SDET (测试开发工程师)。请在不违反"拒绝过度工程化"原则的前提下，审查项目的测试体系，并为**核心用户路径**补强测试。
    - ### 1. 现状评估
        - 检查测试框架是否存在（Jest / Vitest / Playwright / Cypress 等）。
        - 运行现有测试，统计：
            - 通过率
            - 覆盖率（若可获取）
            - 是否有跳过的测试 (`skip`) 或注释掉的测试
    - ### 2. 测试质量审查
        - 是否存在**假阳性**测试（永远通过但无实际断言）？
        - 是否有测试过度依赖实现细节（如测试组件内部 state 变量名），导致重构时脆断？
        - **修正或删除不合格的测试。**
    - ### 3. 核心路径补强（按优先级）
        - **仅对以下核心路径补充测试**，不追求覆盖率数字，不测试 trivial getter/setter：
            - **快乐路径**：用户完成主要任务的主流程（如：新建笔记 → 编辑 → 保存 → 显示）
            - **关键边界**：空状态页面、最大输入长度、网络失败时的降级展示
            - **数据安全**：数据迁移脚本的输入输出验证
    - ### 4. 测试可维护性
        - 提取常用测试夹具 (fixtures) 和工厂函数，避免测试文件中散落硬编码数据。
        - 确保测试描述使用业务语言（而非技术实现语言）。
    - ### 5. CI 集成检查
        - 确认 `npm test` 是否在 CI pipeline 中运行。
        - 若没有 CI 配置文件，创建基础配置（如 `.github/workflows/ci.yml` 仅含 install + lint + test）。
    - ### 6. 输出测试改进摘要
        - 新增测试数
        - 覆盖的核心路径
        - 删除的有害测试数
